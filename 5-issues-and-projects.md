## 5. Reporting & Discussing (Issues & Projects)

To track *what* we need to do and *who* is doing it, we use **Issues** and **Projects**.

* **Issues:** These are essentially task tickets or bug reports. They physically live inside the specific Repository they belong to. 
* **Projects:** We group Issues together into **Projects** (like a task board). This gives us a nice, centralized place so you always know exactly what tasks are assigned to you and the progress of the team.

### 💡 The Magic of the `#` Symbol
Whenever you want to mention a specific issue or PR—whether you are writing a comment, chatting in a discussion, writing a commit message, or filling out a PR description—all you have to do is type the `#` symbol followed by the number (e.g., `#123`). 

GitHub will automatically create a clickable link right to that issue/PR, and notify anyone involved. It’s the easiest way to keep our conversations connected!

***

### 🚀 Pro-Tip Summary of Commands:
```bash
# 1. Get the code
git clone <url>

# 2. Start working safely
git switch -c <branch-name>

# 3. Stage & Snapshot your work
git add .
git commit -m "Description of changes"

# 4. Pull the latest code to be safe
git pull origin main

# 5. Share your work (Use -u for the first push to set upstream!)
git push -u origin <branch-name>
```
